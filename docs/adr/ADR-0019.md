# ADR-0019: NFC normalization dependency for kernel/canon

Status: ACCEPTED — 2026-07-07 (owner acceptance, WP-02.1)
Date: 2026-07-07
Register: D19

## Context

RFC-0001 D2 and docs/vectors/canon.json _rules require strings to be
NFC-normalized before canonical serialization; the golden case
unicode-nfc-normalization feeds NFD input and expects NFC bytes. The Go
standard library has no Unicode normalization. kernel/canon is a
security-critical path: "no new dependencies without an ADR"
(CLAUDE.md). WP-01 cannot ship without resolving this.

## Options

  (a) golang.org/x/text/unicode/norm. Maintained by the Go team,
      quasi-stdlib, table-driven from the Unicode database, no
      transitive dependencies beyond x/text itself. Trade: an external
      module in the most change-averse package; Unicode-version bumps
      of x/text could in principle alter normalization of exotic
      sequences (mitigated: NFC is stable for assigned code points,
      and the vectors + determinism suite pin observable behavior).
  (b) Hand-roll an NFC subset. Trade: thousands of lines of
      table-driven code in a package whose law is "small enough to
      hand-review in full" — strictly worse for security; guaranteed
      divergence on edge cases.
  (c) cgo binding to ICU. Trade: platform-dependent bytes, cgo in the
      kernel, non-hermetic builds. Violates determinism goals outright.

## Decision

ACCEPTED: (a) golang.org/x/text/unicode/norm, pinned in go.mod
(v0.39.0 at adoption), with the canon vectors and determinism property
suite as the behavioral pin. Version bumps of x/text in kernel/canon
require re-running the full conformance gate; a bump that changes any
canonical byte is a frozen-contract change (golden-file law) and is
refused absent its own ADR. As built: x/text v0.39.0 selects its
normalization tables by Go-release build tag — Unicode 15.0.0 under
the current go1.26 toolchain (norm/tables15.0.0.go, Version =
"15.0.0"), switching to Unicode 17.0.0 tables (norm/tables17.0.0.go)
under go1.27+ — so a toolchain upgrade counts as a normalization
version bump for the re-run rule above.

Note: pgregory.net/rapid (property testing, test-only) is used by the
canon determinism suite; it is named as the project's property-test
tool in CLAUDE.md Go conventions, so its adoption is recorded here for
completeness rather than decided.

## Consequences

Locks in: x/text as the single permitted third-party runtime dependency
of kernel/canon; NFC behavior pinned by vectors, not by the library.
Queues: roadmap E5 (cross-check against an independent JCS
implementation) further constrains serialization behavior at Stage-1
exit.
