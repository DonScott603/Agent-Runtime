# ADR-0020: Object-key ordering: UTF-16 code units

Status: ACCEPTED — 2026-07-07 (owner acceptance, WP-02.1)
Date: 2026-07-07
Register: D20

## Context

RFC-0001 D2 says "JCS-style canonicalization (RFC 8785): sorted keys";
docs/vectors/canon.json _rules said only "sorted keys". Every golden
case uses ASCII keys, where the candidate orders agree, so the vectors
did not pin the choice. The WP-01 implementation and its
TestKeyOrderUTF16 already assume UTF-16 code-unit order; the Python
script that originally generated the vectors sorted by code point,
which diverges from UTF-16 order when a key outside the Basic
Multilingual Plane compares against a BMP key above the surrogate
range (e.g. U+10000 vs U+FF61).

## Options

  (a) UTF-16 code-unit order (RFC 8785 §3.2.3). Aligns with actual
      JCS, which roadmap E5 cross-checks against an independent
      implementation at Stage-1 exit; matches the shipped
      implementation. Trade: not the natural Go string order.
  (b) Code-point (= UTF-8 byte) order. Aligns the legacy generation
      script; natural Go string comparison. Trade: guarantees E5
      divergence on non-BMP keys; a private dialect of "JCS-style".

## Decision

(a) UTF-16 code-unit order, per RFC 8785 §3.2.3. Under the golden-file
law, this ADR authorizes: the canon.json _rules amendment stating the
order explicitly, and the new golden case key-order-utf16-astral
(U+10000 sorts before U+FF61). The case's expected bytes and hash were
computed by an independent throwaway reference script implementing the
_rules, not by the implementation under test.

## Consequences

Locks in: key order is now vector-pinned; the legacy generation script
is unsafe as a reference for non-ASCII keys (warning added to
docs/vectors/README.md). The implementation needed no change.

NOTE (nil-vs-empty envelope normalization, recorded here per owner
instruction rather than as a separate ADR): a logical event must have
exactly one canonical byte form, so kernel.NormalizeEnvelope
normalizes nil Blobs to an empty slice and nil Payload to the empty
JSON object before canonicalization; SealEvent (WP-04) must normalize
first. Pinned by TestNormalizeEnvelopeNilVsEmpty.
